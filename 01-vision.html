DESIGN LAWS

不可妥协的设计原则

这些不是 "可以商量", 是 系统的脊梁。任何违反这 5 条的功能, 都不进终态。

"不可能出 bug" — URL 是 source of truth, 不存 React state, AI 自由组合, 失败 fallback 兜底
内容是主, 能力是 buff — 你在 "看见自己", 不在 "训练 capability", 能力藏在学习卡里
AI 决定 UI — 不用固定组件, 让 AI 自由组合 (emoji/颜色/size/位置)
信息 × 空间适配 — AI 决定每张卡占多大 (hero/card/link), 不是固定网格
失败 fallback — 8 种 renderer + safeInteractionType, 任何 prompt 出问题都给简单回退
NOT IN THE END STATE

不在终态里的

这些是看起来 "该有" 但会破坏关系的东西, 终态里 明确不包含。

复杂数据 dashboard
排行榜 / 进度条 / 打卡日历
AI 评分 ("你得了 92 分")
群组 / 社交 / 关注
推送通知 ("今天学了吗")
任务完成度百分比
成就徽章 / 解锁特效
连续打卡 streak 数字

这个系统让你能 "我跟自己说话",
不是 "我跟 AI 说话"。

AI 是 镜子, 不是老师。
镜子帮你看见自己, 不教你怎么做。
你看见得越清楚, 越不需要镜子。
这是终态。
